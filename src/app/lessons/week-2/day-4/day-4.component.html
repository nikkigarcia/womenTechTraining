Assignments

For each assignment listed below:

  • Create a new Angular project (unless stated otherwise).
  • Push up to Github before making any changes to the project.
  • Create a new branch (give appropriate name) and complete the instructions outlined in that branch.
  • Explain why you chose the approach you took? Which problems did you encounter? (add these to a Readme file).
  • Create a Pull Request between your branch and master.
  • Share the link to your PR with me or tag me as a reviewer (GH username: pratimasakinala).

Practicing Built-in Directives

Practice 1:

  • Add a button which says 'Display Details'.
  • Add a paragraph with any content of your choice.
  • Toggle the displaying of that paragraph with the button created in the first step.
  • When the paragraph is displayed, change the button text to 'Hide Details'.
  • Log all button clicks in an array and output that array below the secret paragraph (maybe log a timestamp or simply an incrementing number).
  • Starting at the 10th log item, give all future log items a green background (via ngClass) and white color (ngStyle).

Practice 2:

  • Continue working on the monster-fight project.
  • Create a new branch.
  • Add a new input that binds to hero's name key.
  • Add a button which may only be clicked if the hero's name is NOT an empty string.
  • Once this button is clicked, generate random power for hero and remove/hide both the input and button (added in the previous two steps) from the DOM.
  • Looping though the array of monsters, style the display message such that if the monster's power is greater than hero's, add danger styling.

Practicing Components & Directives

Practice 1:

  • Continue working on the monster project.
  • Create a new branch.
  • Break down the AppComponent into smaller Components.
  • Pass necessary property/event into and out of these new Components.

Additional challenge:

  • Add a 'Fight' button next to each monster message.
  • Upon clicking this button, do Math calculations to see if the hero survived the fight.
  • If the hero dies, clear the monster list and display 'You Lost!' message and bring back the input for hero and reset the game.
  • If the hero survives and the monster has:
    • no more power, remove the monster from the list
    • some power left, keep the monster in the list
    and let the player continue.
  • Make any style changes you need/want to.

Practice 2:

  • Create three new components(manually or with CLI): GameControl, Odd and Even.
  • The GameControl Component should have buttons to start and stop the game.
  • When starting the game, an event (holding a incrementing number) should get emitted each second.
  • The event should be listenable from outside the component.
  • When stopping the game, no more events should get emitted.
  • A new Odd component should get created for every odd number emitted, the same should happen for the Even Component (on even numbers).
  • Simply output Odd - NUMBER or Even - NUMBER in the two components.
  • Style the element holding your output text differently in both components.

Practicing Databinding

Practice 1 (Passing Data to nested component):

  • Create two components: ParentComponent (manually) and ChildComponent (using CLI).
  • In ParentComponent, add a header (h1) with text "I'm a Container Component".
  • Display ChildComponent below this header.
  • In ChildComponent, create a property title and set it to "I'm a nested component".
  • Display a header (h2) with the value of title in ChildComponent template (use inline template).
  • Run the project to make sure the components load with proper data.
  • Modify ChildComponent such that the title property is no longer hard-coded to the string value. But it takes any value passed into it from ParentComponent.
  • Create a childTitle property in ParentComponent and set it to "Text passed to child".
  • Update the ParentComponent template to bind to title property of ChildComponent and pass in childTitle property into it.

Practice 2 (Passing Data from nested component):

  • Continue working on the same project that was created in Practice 1 above.
  • Add a button in ChildComponent below the header with text "Click me!".
  • On clicking this button, execute a method notifyParent that emits the event to ParentComponent.
  • In ParentComponent template, capture the event emitter and execute a method onNotify (which prints the message in console).
