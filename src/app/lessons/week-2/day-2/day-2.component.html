Practicing Databinding

Practice 1 (String Interpolation):

  • Create a heading property in AppComponent and set it to "Person Details".
  • Create a person property of type any.
  • Set person to be an object consisting of name (empty object) and gender (empty string) properties.
  • The name property in person should have first and last properties (both of type string and set to some default values).
  • In the template, display the heading followed by a paragraph.
  • Output below message in the paragraph-
    Name:          
    City:          
  • In the Name section in the paragraph above, display 'Mr' or 'Mrs' based on the gender of the person. (for example: Mr John Doe)
  • In the City section, check if person property has a address property. If it exists, display its value. Otherwise leave the text blank.

Practice 2 (String Interpolation):

  • Create two properties (first and second) of type number in AppComponent and set some default values.
  • Output these properties in the template.
  • Output the sum of these properties in the template (Sum of numbers: ___).
  • Output the largest of these property values in the template (Largest of numbers: ___).

Practice 3 (Property Binding):

  • Create a class Person (fields: name, gender and rating).
  • Set appropriate types to above fields.
  • Create two variable female and male.
  • Set these variables to be instances of Person class (initialize with appropriate values (gender: 'f' for female) and (gender: 'm' for male)).
  • Create a person property.
  • Create a method that randomly pick between female and male variables and sets it to person property.
  • In the template:
    • Use property binding to set an h1 tag's content (textContent) to be "Name: ___ ". Fill in the blank with person's name.
    • Add two buttons below the h1. One should say "Male" and other "Female".
    • These buttons should be disabled if person's gender is male or female respectively.
    • On clicking the buttons, person should be set to the opposite gender.
    • (For example, if person is originally set to female, only Male button should be enabled. And on clicking this button, person should be set to male. And hence the button should be disabled.)

Practice 4:

  • Call this project monster-fight.
  • Work on each of below sections in separate branches (appropriately named).

  • String Interpolation:

    • Create a monsterName property of type string in AppComponent.
    • Output the monsterName property in a paragraph.

  • Property Binding:

    • Add a button (below the paragraph) which may only be clicked if the monsterName is NOT an empty string.
    • Upon clicking the button, generate a random power value for the monster and store it in a property (appropriately named).
    • Display a message (with monsterName and power included) below the button for all monsters created.
    • Test functionality by adding/removing value of monsterName property.

Practicing Built-in Directives

Practice 1:

  • Add a button which says 'Display Details'.
  • Add a paragraph with any content of your choice.
  • Toggle the displaying of that paragraph with the button created in the first step.
  • When the paragraph is displayed, change the button text to 'Hide Details'.
  • Log all button clicks in an array and output that array below the secret paragraph (maybe log a timestamp or simply an incrementing number).
  • Starting at the 10th log item, give all future log items a green background (via ngClass) and white color (ngStyle).

Practice 2:

  • Continue working on the monster-fight project.
  • Create a new branch.
  • Add a new input that binds to hero's name property.
  • Add a button which may only be clicked if the hero's name is NOT an empty string.
  • Once this button is clicked, generate random power for hero and remove/hide both the input and button (added in the previous two steps) from the DOM.
  • Looping though the array of monsters, style the display message such that if the monster's power is greater than hero's, add danger styling.

Practicing Components & Directives

Practice 1:

  • Continue working on the monster project.
  • Create a new branch.
  • Break down the AppComponent into smaller Components.
  • Pass necessary property/event into and out of these new Components.

Additional challenge:

  • Add a 'Fight' button next to each monster message.
  • Upon clicking this button, do Math calculations to see if the hero survived the fight.
  • If the hero dies, clear the monster list and display 'You Lost!' message and bring back the input for hero and reset the game.
  • If the hero survives and the monster has:
    • no more power, remove the monster from the list
    • some power left, keep the monster in the list
    and let the player continue.
  • Make any style changes you need/want to.

Practice 2:

  • Create three new components(manually or with CLI): GameControl, Odd and Even.
  • The GameControl Component should have buttons to start and stop the game.
  • When starting the game, an event (holding a incrementing number) should get emitted each second.
  • The event should be listenable from outside the component.
  • When stopping the game, no more events should get emitted.
  • A new Odd component should get created for every odd number emitted, the same should happen for the Even Component (on even numbers).
  • Simply output Odd - NUMBER or Even - NUMBER in the two components.
  • Style the element holding your output text differently in both components.
