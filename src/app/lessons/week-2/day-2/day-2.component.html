Practicing Databinding

Practice 1 (String Interpolation):

  • Create a heading property in AppComponent and set it to "Person Details".
  • Create a person property of type any.
  • Set person to be an object consisting of name (empty object) and gender (empty string) keys.
  • The name key in person should have first and last keys (both of type string and set to some default values).
  • In the template, display the heading followed by a paragraph.
  • Output below message in the paragraph-
    Name:          
    City:          
  • In the Name section in the paragraph above, display 'Mr' or 'Mrs' based on the gender of the person. (for example: Mr John Doe)
  • In the City section, check if person property has a address key. If it exists, display its value. Otherwise leave the text blank.

Practice 2 (String Interpolation):

  • Create two properties (first and second) of type number in AppComponent and set some default values.
  • Output these properties in the template.
  • Output the sum of these properties in the template (Sum of numbers: ___).
  • Output the largest of these property values in the template (Largest of numbers: ___).

Practice 3 (Property Binding):

  • Create a class Person (fields: name, gender and rating).
  • Set appropriate types to above fields.
  • Create two properties female and male.
  • Set these properties to be instances of Person class (initialize with appropriate values (gender: 'f' for female) and (gender: 'm' for male)).
  • Create a person property.
  • Create a method that randomly pick between female and male properties and sets it to person property.
  • In the template:
    • Use property binding to set an h1 tag's content (textContent) to be "Name: ___ ". Fill in the blank with person's name.
    • Add two buttons below the h1. One should say "Male" and other "Female".
    • These buttons should be disabled if person's gender is male or female respectively.
    • On clicking the buttons, person should be set to the opposite gender.
    • (For example, if person is originally set to female, only Male button should be enabled. And on clicking this button, person should be set to male. And hence the button should be disabled.)

Practice 4:

  • Call this project monster-fight.
  • Work on each of below sections in separate branches (appropriately named).

  • String Interpolation:

    • Create a monsterName property of type string in AppComponent.
    • Output the monsterName property in a paragraph.

  • Property Binding:

    • Add a button (below the paragraph) which may only be clicked if the monsterName is NOT an empty string.
    • Upon clicking the button, generate a random power value for the monster and store it in a property (appropriately named).
    • Display a message (with monsterName and power included) below the button for all monsters created.
    • Test functionality by adding/removing value of monsterName property.
