Assignments

For each assignment listed below:

  • Create a new Angular project (unless stated otherwise).
  • Push up to Github before making any changes to the project.
  • Create a new branch (give appropriate name) and complete the instructions outlined in that branch.
  • Explain why you chose the approach you took? Which problems did you encounter? (add these to a Readme file).
  • Create a Pull Request between your branch and master.
  • Share the link to your PR with me or tag me as a reviewer (GH username: pratimasakinala).

Merge Recipe-book branches

  • Working with a partner, merge all branches in recipe-book into master in proper order.
  • Remember to rebase and push to already pushed branches with force flag.
  • Take a look at the order of topics covered to determine the order of branches to be merged.

Practice Services

Practice 1:

  • Create a invention model with name, invention and year properties.
  • Create an inventions property in AppComponent that can store an array of inventions.
  • Add some test data to inventions array.
  • Create 3 inputs in AppComponent template to set name, invention and year properties in AppComponent.
  • Add a button below the inputs.
  • On clicking this button, save the input values as a new instance of Invention model and push to the inventions array.
  • Under the button, display the list of all inventions.

Practice 2:

  • Continue working on above project, create a new branch off of practice 1 branch.
  • Create a InventionService and move inventions list to this service for central data management.
  • Make this list private so it is not accessible from outside directly.
  • Create a getter method called getInventions that returns a copy of inventions array.
  • Optimize the app by using this new service.

Additional Challenge: Practice Routing

  • Working on original recipe-book application, create a practice-routing branch (branch off of services branch).
  • Create basic route path configurations for the application.
  • Use this route configuration for navigating between recipe and shopping-list features.
