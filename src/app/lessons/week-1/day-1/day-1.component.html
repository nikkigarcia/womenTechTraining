What is Angular?

Angular is a Typescript-based open-source front-end web application framework that allows you to create reactive SPAs.

What are SPAs?

Single Page Applications (SPAs) are web applications where the page never changes but is rewritten. There is only one HTML file and a bunch of JS code we get from the server. Every change is rendered in the browser. This approach avoids interruption of user experience between successive pages making application behave more like a desktop application.

Angular Framework

There are various versions of Angular framework available.

Angular 1 was the first Angular framework (released in 2009). It was the first to make single-page-applications and better way of making DOM (Document Object Model) manipulations popular back in the day.

Angular 2 (2015-2016) was a complete rewrite of Angular 1 to fix many issues mainly performance issues.

Angular 4, 5 and 6 are the latest versions of Angular 2. So in general, we refer to Angular 2 and higher as just "Angular". Angular 1, however, is not related to these version. Therefore Angular 1 is called "AngularJS".

Currently 6 is the latest version. It is not a complete rewrite for every version. These are incremental improvements of Angular 2.

Angular CLI

Angular projects are more elaborate in regards to their build workflow, there are couple of files that need to be converted before they can run in browser. Angular CLI (Command Line Interface) is the recommended and best way of creating Angular projects as it does all of that work for us and heavily optimizes our code.


In order to make use of Angular CLI, we need to make sure we have NodeJS, npm and CLI installed globally on our machine.


  • Installing or Updating NodeJS -
    Go here and download the latest version

  • Installing or Updating npm -
    Run [sudo] npm install -g npm (sudo maybe required for Mac/Linux)

  • Installing the CLI -
    Run npm install -g @angular/cli/@latest

  • Creating a new project -

    Open your terminal/command line and navigate into a folder where you want to create the new project using the cd command.

    Run ng new my-first-app (project name cannot be test as it is a reserved word).

    Navigate into the project. cd my-first-app

    Run ng serve (to bring up a development server to run your build so you can see it in the browser)

    Go to http://localhost:4200/ to see the default angular application the CLI created.

Typescript

We have come across the word Typescript before in the definition of Angular. But what does it mean?

Typescript is a superset of Javascript. It primarily provides optional static typing, classes and interfaces. It compiles to idiomatic (normal) JavaScript, can dramatically improve your productivity by enabling rich tooling experiences, all while maintaining your existing code and continuing to use the same JavaScript libraries you already love. One of the big benefits is to enable IDEs to provide a richer environment for spotting common errorsÂ as you type the code.

TypeScript makes it easier to write cross-platform, application scale, JavaScript that runs in any browser or in any host.

Types:

The Type system represents the different types of values supported by the language. It checks for the validity of the supplied values, before they are stored or manipulated by the program. It further allows for richer code hinting and automated documentation too.

Built-in types -

  • Number - Can be used to represent both integers and floats.
  • String - Represents a sequence of Unicode characters.
  • Boolean - Represents logical values (ture or false).
  • Null - Represents an intentional absence of an object value.
  • Undefined - Denotes value given to all uninitialized variables.

Null and undefined - Are they the same?

Class:

A class defines a blueprint of what an object should look like and act like and then implements that blueprint by initialising class properties and defining methods.

A class definition can include the following -

  • Fields - A field is any variable declared in a class. Fields represent data pertaining to objects.
  • Constructors - Responsible for allocating memory for the objects of the class.
  • Functions - Functions represent actions an object can take. They are also at times reffered to as methods.

These components put together are termed as the data members of the class.

Inheritance:

A derived class that derives from the base class uses the extends keyword. Derived classes are often called subclasses, and base classes are often called superclasses.

Interface

An interface is a group of related properties and methods that describe an object, but neither provides implementation nor initialisation for them. Unlike classes, an interface is a virtual structure that only exists within the context of TypeScript. The TypeScript compiler uses interfaces solely for type-checking purposes.

Generics

With generics, TypeScript enables you to write code that can act on a variety of data types instead of being limited to a single one. Why do we need them at all?

Modules:

The word modules refers to small units of independent, reusable code that exports specific objects, making them avaibale for other modules to require in their programs. Modules are highly self-contained with distict functionality, allowing them to be shuffled, removed, or added as necessary, without disrupting the system as a whole. They provide the possibility to group related logic, encapsulate it, structure your code and prevent pollution of the global namespace.

Components

Components are key features in Angular. We build our application by composing a bunch of components. We start with AppComponent (root component) which holds our entire application.

A component consists of the following -

  • Template - contains the HTML that needs to be rendered in the view for the application. Also includes binding and directives.
  • Class - contains properties and methods. This has the code which is used to support the view. It is defined in Typescript.
  • Metadata - has the extra data defined for the class. It is defined with a decorator.
