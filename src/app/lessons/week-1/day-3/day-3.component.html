Practicing Typescript

Practice 1 (Class & Class Inheritance):

  • Create a class Point.
  • Define x and y fields of type number.
  • Define a constructor that sets these fields.
  • Create a method add that takes in an argument (newPoint) of type Point.
  • add method should return a new Point generated having x value to be the sum of existing point's x and newPoint's x. Same with the y value.
  • Create two variables p1 and p2 which are instances of Point class.
  • Create a third variable p3 which adds p2 to p1 (p1.add(p2)). Print this variable's value to the console.
  • Create a derived class Point3D (base class = Point).
  • Add z field of type number.
  • Setup the constructor to set x and y fields in the base class and z field.
  • Create a method add that takes in an argument (new3DPoint) of type Point3D.
  • In this method, add respective fields of existing point3D and new3DPoint and return the value.
  • Create two variables p3D1 and p3D2 which are instances of Point3D class.
  • Create a third variable p3D3 which adds p3D2 to p3D1. Print this variable's value to the console.

Practice 2 (Interface):

  • Create an interface Person.
  • Define firstName, lastName fields and sayHi method, all of string type.
  • Create a customer variable that is of Person type.
  • Set some values to the fields. Set sayHi method to return "Hi there" message.
  • Create an employee variable that is also of type Person.
  • Set some values to the fields with sayHi returning "Hello!!" message.
  • Print both customer and employee variables to the console.

Practicing Components

  • Create three new Components: WarningMessage (manually), SuccessMessage (using CLI) and DangerMessage (either).
  • For WarningMessage Component:
    • use inline template and external styles.
    • use default selector.
  • For SuccessMessage Component:
    • use external template and inline styles.
    • use attribute selector.
  • For DangerMessage Component:
    • use external or internal template and styles.
    • use class selector.
  • Output them next to each other in the AppComponent.
  • Output an appropriate warning, success or danger message in these Components.
  • Style the Components appropriately.

Feel free to create more components, nest them into each other!

Additional challenge: Add a button in AppComponent that randomly picks which component to display.
