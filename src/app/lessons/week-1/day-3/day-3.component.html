Practicing Typescript

Practice 1 (Class & Class Inheritance):

  • Create a class Point.
  • Define x and y fields of type number.
  • Define a constructor that sets these fields.
  • Create a method add that takes in an argument (newPoint) of type Point.
  • add method should return a new Point generated having x value to be the sum of existing point's x and newPoint's x. Same with the y value.
  • Create two variables p1 and p2 which are instances of Point class.
  • Create a third variable p3 which adds p2 to p1. Print this variable's value to the console.
  • Create a derived class Point3D (base class = Point).
  • Add z field of type number.
  • Setup the constructor to set x and y fields in the base class and z field.
  • Create a method add that takes in an argument (new3DPoint) of type Point3D.
  • In this method, add respective fields of existing point3D and new3DPoint and return the value.
  • Create two variables p3D1 and p3D2 which are instances of Point3D class.
  • Create a third variable p3D3 which adds p3D2 to p3D1. Print this variable's value to the console.

Practice 2 (Interface):

  • Create an interface Person.
  • Define firstName, lastName fields and sayHi method, all of string type.
  • Create a customer variable that is of Person type.
  • Set some values to the fields. Set sayHi method to return "Hi there" message.
  • Create an employee variable that is also of type Person.
  • Set some values to the fields with sayHi returning "Hello!!" message.
  • Print both customer and employee variables to the console.

Practicing Components

  • Create three new Components: WarningMessage (manually), SuccessMessage (using CLI) and DangerMessage (either).
  • For WarningMessage Component:
    • use internal template and external styles.
    • use default selector.
  • For SuccessMessage Component:
    • use external template and inline styles.
    • use attribute selector.
  • For DangerMessage Component:
    • use external or internal template and styles.
    • use class selector.
  • Output them next to each other in the AppComponent.
  • Output an appropriate warning, success or danger message in these Components.
  • Style the Components appropriately.

Feel free to create more components, nest them into each other!

Additional challenge: Add a button in AppComponent that randomly picks which component to display.

Practicing Databinding

Practice 1 (String Interpolation):

  • Create a heading property in AppComponent and set it to "Person Details".
  • Create a person property of type any.
  • Set person to be an object consisting of name (empty object) and gender (empty string) properties.
  • The name property in person should have first and last properties (both of type string and set to some default values).
  • In the template, display the heading followed by a paragraph.
  • Output below message in the paragraph-
    Name:          
    City:          
  • In the Name section in the paragraph above, display 'Mr' or 'Mrs' based on the gender of the person. (for example: Mr John Doe)
  • In the City section, check if person property has a address property. If it exists, display its value. Otherwise leave the text blank.

Practice 2 (String Interpolation):

  • Create two properties (first and second) of type number in AppComponent and set some default values.
  • Output these properties in the template.
  • Output the sum of these properties in the template (Sum of numbers: ___)
  • Output the largest of these property values in the template (Largest of numbers: ___)

Practice 3 (Property Binding):

  • Create a class Person (fields: name, gender and rating).
  • Set appropriate types to above fields.
  • Create two variable female and male.
  • Set these variables to be instances of Person class (initialize with appropriate values (gender: 'f' for female) and (gender: 'm' for male)).
  • Create a person property.
  • Create a method that randomly pick between female and male variables and sets it to person property.
  • In the template:
    • Use property binding to set an h1 tag's content (textContent) to be "Name: ___ ". Fill in the blank with person's name.
    • Add two buttons below the h1. One should say "Male" and other "Female".
    • These buttons should be disabled if person's gender is male or female respectively.
    • On clicking the buttons, person should be set to the opposite gender.
    • (For example, if person is originally set to female, only Male button should be enabled. And on clicking this button, person should be set to male. And hence the button should be disabled.)
